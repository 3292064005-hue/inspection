# Runtime Validation Evidence

This directory is the mandatory release-time evidence root referenced by
`release/runtime_validation_matrix.yaml`.

Rules:

1. Each required non-simulation scenario must have one JSON evidence file named
   `<scenario-id>.json`.
2. Evidence files must be generated from the matching template in
   `release/runtime_validation_evidence/templates/` and then filled with real
   execution data.
3. Template placeholders must be removed before release validation.
4. `passed: true` is accepted only when every expected check, artifact, and
   failure criterion field is populated with real values.

Bootstrap command:

```bash
python3 scripts/bootstrap_runtime_validation_evidence.py --workspace-root .
```

5. `gate_status.json` is the strict/relaxed gate execution artifact generated by `scripts/run_runtime_validation_matrix.py`.
6. `audit_summary.json` is a derived view generated by `scripts/render_runtime_validation_audit.py`; it is not an independent source of truth.
